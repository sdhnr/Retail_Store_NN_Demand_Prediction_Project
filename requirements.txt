# Project Steps: Retail Inventory Neural Network Model

This document explains the full notebook workflow used to build the Neural Network model for predicting daily `Units Sold`.

## Step 1: Import Libraries

The notebook begins by importing the required Python libraries.

Main libraries used:

* `pandas` for loading and preparing the dataset
* `numpy` for numerical operations
* `matplotlib` for visualization
* `train_test_split` tools and preprocessing tools from scikit-learn
* `MinMaxScaler` for scaling numerical variables
* `OneHotEncoder` for categorical encoding
* `ColumnTransformer` for applying different transformations to different columns
* `MLPRegressor` for building the Neural Network regression model
* `mean_absolute_error`, `mean_squared_error`, and `r2_score` for model evaluation

## Step 2: Load the Dataset

The raw dataset was loaded from:

```text
retail_store_inventory.csv
```

The dataset contains daily inventory and sales records for retail stores and products.

A copy of the raw dataset was created so that the original data remained unchanged.

## Step 3: Convert the Date Column

The `Date` column was converted into datetime format.

This allowed the project to extract useful time-related features later in the notebook.

## Step 4: Isolate Five Rows for Final Testing

Five observations were removed from the dataset before preprocessing and model training.

These five rows were saved separately as:

```text
isolated_5_rows.csv
```

The purpose of isolating these rows was to create a final test set that the model had never seen during training or regular testing.

The isolated rows were not used during:

* preprocessing fitting
* scaling
* encoding
* train-test splitting
* Neural Network training
* regular model evaluation

This step was required by the assignment.

## Step 5: Inspect the Modeling Dataset

After removing the five isolated rows, the remaining dataset was inspected.

The inspection included:

* dataset shape
* column names
* data types
* missing values
* duplicate rows
* summary statistics
* negative-value checks
* categorical-value checks

The dataset had no missing values and no duplicate rows.

## Step 6: Clean Negative Demand Forecast Values

The inspection showed that the `Demand Forecast` column contained negative values.

Negative demand forecasts are not meaningful for inventory planning, so they were replaced with zero using a fixed business rule.

The target variable, `Units Sold`, was not modified.

## Step 7: Feature Engineering

The raw `Date` column was converted into useful time-based features:

* `Date_Year`
* `Date_Month`
* `Date_DayOfWeek`

The original `Date` column was then removed.

The `Units Ordered` column was also removed from the first model because it represents a replenishment decision. The goal of the model is to predict product demand before making stock-allocation and ordering decisions.

## Step 8: Select Target and Predictors

The target variable was:

```text
Units Sold
```

This is the numerical value the model tries to predict.

The predictor variables included inventory, forecast, price, discount, holiday/promotion status, competitor pricing, product information, store information, region, weather, seasonality, and date-based features.

## Step 9: Identify Numerical and Categorical Features

The model used two groups of input variables.

### Numerical Features

* `Inventory Level`
* `Demand Forecast`
* `Price`
* `Discount`
* `Holiday/Promotion`
* `Competitor Pricing`

### Categorical Features

* `Store ID`
* `Product ID`
* `Category`
* `Region`
* `Weather Condition`
* `Seasonality`
* `Date_Month`
* `Date_DayOfWeek`

## Step 10: Chronological Train-Test Split

The data was sorted by date and split chronologically.

The earlier 80% of the data was used for training.

The most recent 20% of the data was used for testing.

A chronological split was selected because the business goal is to predict future-style product demand using historical information.

The five isolated rows were not included in either the training or testing dataset.

## Step 11: Preprocess Input Features

The input variables were preprocessed using a `ColumnTransformer`.

Numerical features were scaled to the 0–1 range using:

```python
MinMaxScaler()
```

Categorical features were converted into dummy variables using:

```python
OneHotEncoder()
```

The preprocessing transformer was fitted only on the training data. This prevented information from the testing dataset from influencing model development.

The fitted transformer was then applied to the testing data.

After preprocessing, the model had:

```text
67 processed input features
```

## Step 12: Scale the Target Variable

The target variable, `Units Sold`, was also scaled to the 0–1 range before training.

A separate `MinMaxScaler` was used for the target variable.

The scaler was fitted only on the training target values.

After prediction, the model outputs were converted back into the original unit scale so the results could be interpreted as units sold.

## Step 13: Build the Neural Network Model

A `MLPRegressor` model was selected because the project predicts a numerical value.

The model architecture was:

| Layer        | Description           |
| ------------ | --------------------- |
| Input Layer  | 67 processed features |
| Hidden Layer | 46 nodes              |
| Output Layer | 1 node                |

The hidden layer size was selected using the guideline:

```text
(2/3 × number of input nodes) + number of output nodes
```

Calculation:

```text
(2/3 × 67) + 1 ≈ 46
```

The model used:

* one hidden layer
* 46 hidden nodes
* ReLU activation function
* Adam optimizer
* early stopping

## Step 14: Train the Neural Network

The Neural Network was trained using:

* preprocessed training inputs
* scaled training target values

Early stopping was enabled. This allowed the model to stop training when improvement became very small.

The model completed training after:

```text
35 iterations
```

The final training loss was approximately:

```text
0.000182
```

## Step 15: Plot the Training Loss Curve

A loss curve was created to show how the model’s training loss changed over the training iterations.

The curve decreased sharply at the beginning and then became almost flat. This showed that the model learned the main patterns quickly and then stopped improving significantly.

## Step 16: Evaluate the Model on the Testing Dataset

The trained model was evaluated using the regular 20% testing dataset.

The testing records were not used during model training.

The scaled predictions were converted back into the original `Units Sold` scale.

The model was evaluated using:

* Mean Absolute Error
* Root Mean Squared Error
* R² Score

### Testing Results

| Metric   |     Result |
| -------- | ---------: |
| MAE      | 7.73 units |
| RMSE     | 9.05 units |
| R² Score |     0.9930 |

The MAE shows that the model’s predictions differed from the actual sales values by approximately 7.73 units on average.

The R² score shows that the model explained approximately 99.3% of the variation in `Units Sold`.

## Step 17: Test the Five Isolated Rows

The five isolated rows were loaded from:

```text
isolated_5_rows.csv
```

Their actual `Units Sold` values were stored separately.

The same feature-engineering rules were applied to the isolated rows.

The target variable was removed before prediction.

The already-fitted preprocessing transformer was applied using:

```python
preprocessor.transform()
```

```python
nn_model.predict()
```
